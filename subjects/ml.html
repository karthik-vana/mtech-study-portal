Q1(a) Define a learning system in the context of machine learning and explain its components. 7M ▼
📖 Definition

A learning system is a computational system that improves its performance on a specific task through experience, using data-driven methods to map inputs to outputs without being explicitly programmed for every scenario.

⚙️ Core Components
  • Task (T): The problem being solved (classification, regression, clustering).
  • Performance Measure (P): Metric to evaluate success — accuracy, MSE, F1-score.
  • Experience (E): Training data from which the model learns patterns.
  • Learner / Model: The algorithm mapping input X to output Y (hypothesis h: X→Y).
  • Feedback Mechanism: Error signal used to update model parameters (loss function).
  • Hypothesis Space (H): Set of all functions the learner can represent.
  • Generalization: Ability to perform correctly on unseen data beyond training set.
📐 Diagram — Learning System Architecture
┌──────────────┐ ┌──────────────────┐ ┌──────────────┐ │ Environment │────▶│ Training Data │────▶│ Learner │ │ (Real World)│ │ (X, Y pairs) │ │ (Algorithm) │ └──────────────┘ └──────────────────┘ └──────┬───────┘ │ ┌─────────▼──────────┐ │ Hypothesis h(x) │ │ (Learned Model) │ └─────────┬──────────┘ │ ┌──────────────┐ ┌──────────────────┐ ┌───────▼────────┐ │ New Input │────▶│ Prediction │◀───│ Performance │ │ (Test Data) │ │ ŷ = h(x) │ │ Evaluation (P) │ └──────────────┘ └──────────────────┘ └────────────────┘
✅ Conclusion

A machine learning system is defined by the triplet (T, P, E). Its components work in tandem: experience drives learning, the hypothesis space constrains what is learnable, and the performance measure guides optimization toward a better generalizing model.

Training data is the labeled or unlabeled dataset D = {(x₁,y₁), …, (xₙ,yₙ)} from which the learner infers a hypothesis h ∈ H. It provides the empirical evidence that drives parameter estimation and model selection.

Q2(b) Illustrate how function approximation is used in machine learning algorithms. Provide examples of algorithms that use function approximation techniques. 8M ▼
📖 Function Approximation

Function approximation is the process of learning a function f̂: X→Y that approximates the true unknown target function f*, using a parametric or non-parametric hypothesis selected from hypothesis space H to minimize generalization error.

∑ Mathematical Basis
Target function: f*: X → Y (unknown) Learned function: f̂(x; θ) ≈ f*(x) Loss: L(θ) = (1/n) Σᵢ ℓ(f̂(xᵢ;θ), yᵢ) Optimization: θ* = argmin L(θ) [via gradient descent]
⚙️ Types & Examples
AlgorithmApproximation TypeFunction Form
Linear RegressionParametric, linearf̂(x) = wᵀx + b
Neural NetworksParametric, nonlinearf̂(x) = σ(Wₙσ(…σ(W₁x)))
k-NNNon-parametricf̂(x) = avg(yᵢ for k nearest xᵢ)
Decision TreesPiecewise constantf̂(x) = partition-based rules
SVMKernel-basedf̂(x) = Σ αᵢyᵢK(xᵢ,x) + b
RBF NetworksRadial basisf̂(x) = Σ wᵢ φ(‖x−cᵢ‖)
✅ Conclusion

Function approximation is the unifying framework of supervised ML. The choice of function class (linear vs nonlinear, parametric vs non-parametric) determines model expressiveness, inductive bias, and computational cost.

Q3(b) Explain the concept of version spaces in machine learning and how they are used in problem-solving. 8M ▼
📖 Definition

The Version Space VS(H,D) is the subset of hypothesis space H consistent with all training examples in D — i.e., every h in VS correctly classifies all training instances.

∑ Formal Definition
VS(H,D) = { h ∈ H | ∀(x,c) ∈ D : h(x) = c } Represented by two boundary sets: S = { maximally specific hypotheses } ← most specific G = { maximally general hypotheses } ← most general VS = { h | ∃s∈S, ∃g∈G : g ≤ h ≤ s }
📐 Diagram — Version Space
Hypothesis Space H ┌──────────────────────────────────────────────┐ │ G (Most General) │ │ ┌──────────────────────────────────────┐ │ │ │ Version Space VS │ │ │ │ (all consistent hypotheses) │ │ │ │ ┌──────────────────────┐ │ │ │ │ │ S (Most Specific) │ │ │ │ │ └──────────────────────┘ │ │ │ └──────────────────────────────────────┘ │ │ (inconsistent hypotheses) │ └──────────────────────────────────────────────┘ On + example: S specializes (narrows) On - example: G specializes (narrows) VS ← ∅ → no consistent hypothesis exists
⚙️ Use in Problem Solving
  • Provides a compact representation of all hypotheses consistent with data.
  • Used in Candidate Elimination Algorithm for concept learning.
  • VS collapse to single h → perfect learner on that data.
  • Guides which new examples are most informative (active learning).
  • Detects inconsistency: if S=∅ or G=∅, no consistent concept exists.
✅ Conclusion

Version space provides a principled framework for concept learning by tracking all consistent hypotheses. The S-G boundary representation enables efficient search without enumerating all hypotheses in H.

Q4(a) Describe the candidate elimination algorithm and explain how it can be used in concept learning. 7M ▼
📖 Definition

The Candidate Elimination Algorithm (Mitchell, 1982) searches the version space by maintaining the most general (G) and most specific (S) boundaries, updating them with each training example to eliminate inconsistent hypotheses.

🔧 Algorithm
Initialize: G ← { ? ? ? … ? } // maximally general S ← { ∅ ∅ ∅ … ∅ } // maximally specific For each training example d = (x, c(x)): IF c(x) = + (positive example): Remove from G all hypotheses inconsistent with x For each s in S inconsistent with x: Replace s with all minimal generalizations h s.t.: h is consistent with x, AND ∃g∈G: h ≤ g Remove from S any h less general than another in S IF c(x) = − (negative example): Remove from S all hypotheses inconsistent with x For each g in G inconsistent with x: Replace g with all minimal specializations h s.t.: h is consistent with x, AND ∃s∈S: s ≤ h Remove from G any h more general than another in G Output: S and G representing Version Space
✅ Conclusion

CEA converges to the correct concept if the target is in H and data is noise-free. It is an exhaustive version space search — suitable for small H but computationally intractable for large attribute spaces.

Q4(b) Define inductive bias and explain its importance in machine learning. Discuss how it affects the learning process and model selection. 8M ▼
📖 Definition

Inductive Bias is the set of assumptions a learner uses to predict outputs for inputs not yet encountered in training — it is the prior knowledge that enables generalization beyond the observed data.

⚙️ Types of Inductive Bias
  • Restriction Bias: Limits hypothesis space (e.g., linear classifiers only consider hyperplanes).
  • Preference Bias: Prefers simpler hypotheses among those consistent with data (Occam's Razor).
  • No-Free-Lunch Theorem: Without bias, no learner outperforms another over all possible problems.
📋 Inductive Bias per Algorithm

Naive BayesFeatures are conditionally independent given class
SVMMaximum margin separating hyperplane
Neural NetworksDistributed hierarchical feature representations
✅ Conclusion

Inductive bias is essential — without it, no generalization is possible (No-Free-Lunch). Matching the learner's bias to the true data distribution is the art of model selection and governs whether the model underfits, overfits, or generalizes well.

Q5(a) Describe the decision tree algorithm and explain the steps involved in constructing a decision tree for classification tasks. 7M ▼
📖 Definition

A Decision Tree is a hierarchical, tree-structured model where each internal node tests a feature, each branch represents a feature value, and each leaf node holds a class label — classification is done by traversing root to leaf.

∑ Key Formulas (ID3)
Entropy: H(S) = −Σ pᵢ log₂(pᵢ) Info Gain: IG(S,A) = H(S) − Σ_{v∈Values(A)} (|Sᵥ|/|S|)·H(Sᵥ) Best Attribute: A* = argmax_A IG(S,A)
🔧 ID3 Algorithm
ID3(Examples, Target, Attributes): IF all examples are +ve → Return leaf(+) IF all examples are -ve → Return leaf(-) IF Attributes = ∅ → Return leaf(majority class) A* ← argmax Information_Gain(Examples, A) Create root node with test A* For each value vᵢ of A*: Sᵢ ← examples where A* = vᵢ IF Sᵢ = ∅: Add leaf(majority class of Examples) ELSE: Add subtree ← ID3(Sᵢ, Target, Attributes − {A*}) Return root
📐 Example Tree Structure
[Outlook?] / | \ Sunny Overcast Rainy / | \ [Humidity?] [YES] [Wind?] / \ / \ High Normal Strong Weak | | | | [NO] [YES] [NO] [YES]
✅ Conclusion

Decision trees are interpretable, non-parametric classifiers. ID3 greedily maximizes information gain. Pruning prevents overfitting. Extensions like C4.5 (gain ratio) and CART (Gini) address ID3's bias toward multi-valued attributes.

Q1(a) Apply Bayes' Theorem to update beliefs in a medical diagnosis scenario. 7M ▼
📖 Bayes' Theorem

Bayes' Theorem provides a principled way to update the probability of a hypothesis h given observed data D: P(h|D) = P(D|h)·P(h) / P(D)

∑ Medical Diagnosis Example
Problem: A disease D affects 1% of population. Test has: Sensitivity P(+|D) = 0.99, Specificity P(−|D̄) = 0.95 Given: P(D) = 0.01 → Prior (prevalence) P(D̄) = 0.99 P(+|D) = 0.99 → Likelihood (sensitivity) P(+|D̄) = 1 − 0.95 = 0.05 → False positive rate Step 1 — P(+) [Total Probability]: P(+) = P(+|D)·P(D) + P(+|D̄)·P(D̄) = 0.99×0.01 + 0.05×0.99 = 0.0099 + 0.0495 = 0.0594 Step 2 — Posterior P(D|+): P(D|+) = P(+|D)·P(D) / P(+) = (0.99 × 0.01) / 0.0594 ≈ 0.167 (only ~16.7% likely to have disease!) Insight: Low prevalence → high false positives despite good test.
⚙️ Key Terms
  • Prior P(h): Belief before observing data (disease prevalence).
  • Likelihood P(D|h): Probability of test result given hypothesis (sensitivity/specificity).
  • Posterior P(h|D): Updated belief after observing data.
  • Evidence P(D): Normalizing constant ensuring probabilities sum to 1.
✅ Conclusion

Bayes' Theorem shows that a positive test result with low disease prevalence leads to a low posterior probability — highlighting the critical role of prior beliefs. This principle underpins all Bayesian classifiers and spam filters.

Q2(a) Explain the concept of maximum likelihood estimation (MLE). How is it used to estimate parameters of a probability distribution? 7M ▼
📖 Definition

Maximum Likelihood Estimation (MLE) finds the parameter vector θ that maximizes the probability (likelihood) of observing the given training data D, without assuming any prior distribution over θ.

∑ MLE Formulation
Likelihood: L(θ) = P(D|θ) = Π P(xᵢ|θ) [for i.i.d. data] Log-Likelihood: ℓ(θ) = log L(θ) = Σ log P(xᵢ|θ) MLE Estimator: θ_MLE = argmax_θ ℓ(θ) Example — Gaussian MLE: P(x|μ,σ²) = (1/√2πσ²) exp[−(x−μ)²/2σ²] μ_MLE = (1/n) Σ xᵢ [sample mean] σ²_MLE = (1/n) Σ (xᵢ−μ)² [biased variance]
⚙️ Properties of MLE
  • Consistency: θ_MLE → θ* as n → ∞.
  • Asymptotic Normality: √n(θ_MLE − θ*) → N(0, I⁻¹(θ*)).
  • Efficiency: Achieves Cramér-Rao lower bound asymptotically.
  • No Prior: Unlike MAP, MLE ignores prior knowledge — can overfit with small data.
✅ Conclusion

MLE is the most widely used parameter estimation method. It is statistically principled, computationally tractable, and serves as the foundation for logistic regression, Naive Bayes, and Gaussian mixture models.

Q2(b) Discuss how MDL helps in balancing model complexity and data fit in statistical learning. 8M ▼
📖 MDL Principle

Minimum Description Length (MDL) states that the best hypothesis h is the one that minimizes the total description length: the bits needed to encode h plus the bits needed to encode data D given h.

∑ MDL Formula
h_MDL = argmin_{h∈H} [ L(h) + L(D|h) ] Where: L(h) = bits to encode hypothesis (model complexity) L(D|h) = bits to encode data given hypothesis (data fit) Connection to MAP: L(h) ≡ −log₂ P(h) [prior = complexity penalty] L(D|h) ≡ −log₂ P(D|h) [likelihood = data fit] → MDL ≡ MAP estimation
⚙️ Balancing Complexity vs Fit
  • Underfitting: L(h) small but L(D|h) large → poor data description.
  • Overfitting: L(D|h) ≈ 0 but L(h) very large → memorizes noise.
  • Optimal: MDL finds the sweet spot minimizing total description length.
  • Occam's Razor: MDL formally justifies preferring simpler models.
✅ Conclusion

MDL provides an information-theoretic foundation for model selection, directly embodying Occam's Razor. It is equivalent to Bayesian MAP estimation and provides a rigorous way to prevent overfitting without cross-validation.

Q3(a) Evaluate the performance of the Bayes Optimal Classifier with respect to the practical limitations of real-world data. 7M ▼
📖 Bayes Optimal Classifier

The Bayes Optimal Classifier achieves the lowest possible error rate by selecting the class with maximum posterior probability given the observed instance: ŷ = argmax_c P(c|x).

∑ Formulation
Bayes Optimal Decision: ŷ = argmax_{c ∈ C} P(c|x) = argmax_c P(x|c)·P(c) Combining hypotheses: ŷ = argmax_{c} Σ_{h∈H} P(c|h,x)·P(h|D) Bayes Error Rate (irreducible): ε* = 1 − max_c P(c|x) [minimum achievable error]
⚙️ Practical Limitations
  • Unknown True Distribution: P(x|c) and P(c) must be estimated from finite data → estimation errors.
  • Curse of Dimensionality: Estimating P(x|c) in high dimensions requires exponentially more data.
  • Computational Cost: Summing over all hypotheses in H is intractable for large H.
  • Model Misspecification: Assumed distribution family may not match true distribution.
  • Noisy Labels: Training labels may be incorrect, corrupting likelihood estimates.
✅ Conclusion

The Bayes Optimal Classifier is the theoretical gold standard but is impractical — it requires exact knowledge of the true distribution. Practical classifiers approximate it via parametric assumptions (Naive Bayes) or discriminative approaches (SVM, neural networks).

Q3(b) Describe the Gibbs sampling algorithm and its use in probabilistic inference. 8M ▼
📖 Definition

Gibbs Sampling is a Markov Chain Monte Carlo (MCMC) method for approximating joint distributions P(x₁,…,xₙ) that are difficult to sample directly, by iteratively sampling each variable conditioned on all others.

🔧 Gibbs Sampling Algorithm
Initialize: x⁽⁰⁾ = (x₁⁽⁰⁾, x₂⁽⁰⁾, …, xₙ⁽⁰⁾) [random init] For t = 1, 2, …, T: [burn-in + sampling] x₁⁽ᵗ⁾ ~ P(x₁ | x₂⁽ᵗ⁻¹⁾, x₃⁽ᵗ⁻¹⁾, …, xₙ⁽ᵗ⁻¹⁾) x₂⁽ᵗ⁾ ~ P(x₂ | x₁⁽ᵗ⁾, x₃⁽ᵗ⁻¹⁾, …, xₙ⁽ᵗ⁻¹⁾) … xₙ⁽ᵗ⁾ ~ P(xₙ | x₁⁽ᵗ⁾, x₂⁽ᵗ⁾, …, xₙ₋₁⁽ᵗ⁾) [Update each variable using latest values of others] Discard burn-in samples. Use {x⁽ᵗ⁾}ₜ₌ᵦᵤᵣₙ₋ᵢₙ^T to estimate E[f(x)] ≈ (1/T)Σ f(x⁽ᵗ⁾)
⚙️ Applications in Probabilistic Inference
  • Bayesian posterior inference when direct computation is intractable.
  • Training of Bayesian networks and Hidden Markov Models.
  • Latent Dirichlet Allocation (LDA) for topic modeling.
  • Missing data imputation in probabilistic graphical models.
✅ Conclusion

Gibbs sampling leverages the fact that conditional distributions are often easy to sample even when the joint is intractable. By the ergodic theorem, the Markov chain converges to the true joint distribution, making it a cornerstone of Bayesian computation.

Q4(b) Illustrate the construction of a Bayesian Belief Network for a simple problem, and explain how inference is performed. 8M ▼
📖 Bayesian Belief Network (BBN)

A Bayesian Belief Network is a Directed Acyclic Graph (DAG) where nodes represent random variables and edges represent conditional dependencies. Each node has a Conditional Probability Table (CPT) given its parents.

📐 Example: Rain → Sprinkler → Wet Grass
[Cloudy] / \ [Rain] [Sprinkler] \ / [Wet Grass] CPT — P(Rain|Cloudy): Cloudy=T: P(Rain=T)=0.8 P(Rain=F)=0.2 Cloudy=F: P(Rain=T)=0.2 P(Rain=F)=0.8 CPT — P(WetGrass|Rain,Sprinkler): Rain=T, Spr=T: P(WG=T)=0.99 Rain=T, Spr=F: P(WG=T)=0.90 Rain=F, Spr=T: P(WG=T)=0.90 Rain=F, Spr=F: P(WG=T)=0.00
⚙️ Inference Methods
  • Exact Inference: Variable elimination — sum-out irrelevant variables using marginalization.
  • Belief Propagation: Message passing on tree-structured BBNs (exact for trees).
  • Approximate Inference: MCMC (Gibbs sampling), Variational Bayes for complex networks.
  • Joint Probability: P(C,R,S,W) = P(C)·P(R|C)·P(S|C)·P(W|R,S)
✅ Conclusion

BBNs compactly encode joint distributions via conditional independence, reducing exponential parameters to polynomial. They support both causal reasoning (predictive) and diagnostic reasoning (abductive), making them powerful for medical, fault-diagnosis, and NLP systems.

Q5(a) Explain the Expectation-Maximization (EM) algorithm and its use in parameter estimation for latent variable models. 7M ▼
📖 EM Algorithm

The Expectation-Maximization (EM) algorithm is an iterative method to find MLE parameters in models with latent (hidden) variables Z, alternating between computing expected complete-data log-likelihood (E-step) and maximizing it (M-step).

🔧 EM Algorithm
Initialize: θ⁰ (random or k-means init) Repeat until convergence: E-STEP (Expectation): Compute Q(θ|θᵗ) = E_{Z|X,θᵗ}[log P(X,Z|θ)] i.e., compute responsibility r(zᵢ) = P(zᵢ|xᵢ,θᵗ) M-STEP (Maximization): θᵗ⁺¹ = argmax_θ Q(θ|θᵗ) Update parameters using soft assignments from E-step Until: |L(θᵗ⁺¹) − L(θᵗ)| < ε [convergence criterion] Property: L(θᵗ⁺¹) ≥ L(θᵗ) [monotone improvement]
⚙️ Applications
  • Gaussian Mixture Models (GMM): E-step computes soft cluster assignments; M-step updates μ, Σ, π.
  • Hidden Markov Models (HMM): Baum-Welch algorithm is a special case of EM.
  • Missing Data Imputation: Treat missing values as latent variables.
  • Latent Dirichlet Allocation: Topic modeling via variational EM.
✅ Conclusion

EM guarantees monotone increase in log-likelihood and converges to a local maximum. Its weakness is sensitivity to initialization and convergence to local optima. Multiple restarts are used to mitigate this.

Q1(a) Explain the k-Nearest Neighbor (k-NN) algorithm and how it is used for classification. 7M ▼
📖 Definition

k-Nearest Neighbor (k-NN) is a non-parametric, lazy learning algorithm that classifies a query point by majority vote among its k closest training instances in the feature space, using a distance metric.

🔧 k-NN Classification Algorithm
Training: Store all (xᵢ, yᵢ) pairs — no model fitting! Prediction for query point x_q: 1. Compute d(x_q, xᵢ) ∀ training points [Euclidean: d = √Σ(x_q,j − x_i,j)²] 2. Sort distances; select k nearest neighbors N_k(x_q) 3. Classify by majority vote: ŷ = argmax_c |{xᵢ ∈ N_k : yᵢ = c}| 4. For regression: ŷ = (1/k) Σ_{xᵢ∈N_k} yᵢ
📐 k-NN Decision Boundary
Feature Space: ○ ○ ● ● ○ ●● ● ○ ○ x? ●● ○ ● ● For k=3: x? → 2 neighbors are ●, 1 is ○ → Classify as ● (majority) Decision boundary = Voronoi-based region boundary (Non-linear, adapts to local data structure)
⚡ Properties
  • Time Complexity: Training O(1), Prediction O(n·d) — expensive for large n.
  • k selection: Odd k avoids ties; optimal k via cross-validation.
  • Curse of Dimensionality: Distance loses meaning in high dimensions.
✅ Conclusion

k-NN is simple yet powerful for low-dimensional problems. Its non-parametric nature means it makes no distributional assumptions. The choice of k controls the bias-variance tradeoff: small k → low bias/high variance; large k → high bias/low variance.

📖 RBF Kernel

The Radial Basis Function (RBF) kernel maps input vectors to an infinite-dimensional feature space implicitly, computing similarity as a function of Euclidean distance between points — enabling nonlinear classification without explicit transformation.

∑ RBF Kernel Formula
K(x, xᵢ) = exp(−γ‖x − xᵢ‖²) where γ = 1/(2σ²) Properties: K(x,x) = 1 [self-similarity = 1] K(x,xᵢ) ∈ (0,1] [always positive] K(x,xᵢ) → 0 as ‖x−xᵢ‖ → ∞ [far points = dissimilar] Kernel Matrix (Gram): Kᵢⱼ = K(xᵢ,xⱼ) [PSD matrix]
⚙️ Role in ML Tasks
  • SVM: Enables nonlinear classification by implicitly mapping to infinite-dimensional RKHS.
  • RBF Networks: Activation functions centered at training points — universal approximation.
  • Gaussian Processes: Defines covariance between function outputs.
  • Bandwidth γ: Large γ → sharp, localized influence (overfit); small γ → broad influence (underfit).
✅ Conclusion

RBF kernel is the most widely used kernel in SVMs due to its universal approximation capability and guaranteed positive semi-definiteness. The hyperparameter γ controls the decision boundary complexity and must be tuned via cross-validation.

Q3(a) Explain the concept of Support Vector Machines (SVMs) and their importance in machine learning. 7M ▼
📖 SVM Definition

A Support Vector Machine (SVM) is a discriminative classifier that finds the optimal hyperplane maximizing the margin (geometric separation) between classes, using only the support vectors — the critical boundary data points.

∑ SVM Optimization Problem
Primal (Hard Margin): Minimize: ½‖w‖² Subject to: yᵢ(wᵀxᵢ + b) ≥ 1 ∀i Margin = 2/‖w‖ [maximize by minimizing ‖w‖²] Dual Formulation (QP): Maximize: Σ αᵢ − ½ Σᵢ Σⱼ αᵢαⱼyᵢyⱼ xᵢᵀxⱼ Subject to: Σ αᵢyᵢ = 0, αᵢ ≥ 0 Decision: f(x) = sign(Σ αᵢyᵢ xᵢᵀx + b)
📐 SVM Diagram
● = Class +1 ○ = Class -1 ○ ○ | ● ● ← decision boundary w·x+b=0 ○ | ● ● ○ ○ | ● ←margin→←margin→ w·x+b=-1 w·x+b=+1 (support vectors on dashed lines) Total margin = 2/‖w‖
⚙️ Importance in ML
  • Maximizing margin → better generalization via structural risk minimization.
  • Kernel trick enables nonlinear classification without explicit feature mapping.
  • Sparsity: only support vectors determine the boundary — robust to outliers.
  • Strong theoretical foundation via VC dimension and PAC learning.
✅ Conclusion

SVM achieves maximum margin separation, directly minimizing generalization error via structural risk minimization. It dominated pre-deep-learning benchmarks and remains relevant for small/medium datasets and high-dimensional problems.

Q3(b) Illustrate how the optimal hyperplane is determined in a linearly separable SVM. 8M ▼
📖 Optimal Hyperplane

The optimal hyperplane in a linearly separable SVM is the unique hyperplane w·x + b = 0 that maximally separates the two classes — it lies equidistant from the closest points (support vectors) of each class.

🔧 Derivation Steps
Step 1: Define canonical hyperplanes: w·x + b = +1 (positive class boundary) w·x + b = −1 (negative class boundary) Constraints: yᵢ(w·xᵢ + b) ≥ 1 Step 2: Compute margin: Distance from w·x+b=+1 to w·x+b=−1: margin = 2/‖w‖ Step 3: Formulate QP: min ½‖w‖² s.t. yᵢ(w·xᵢ + b) ≥ 1 Step 4: Lagrangian: L = ½‖w‖² − Σ αᵢ[yᵢ(w·xᵢ+b) − 1] ∂L/∂w = 0 → w = Σ αᵢyᵢxᵢ ∂L/∂b = 0 → Σ αᵢyᵢ = 0 Step 5: KKT conditions: αᵢ ≥ 0; αᵢ[yᵢ(w·xᵢ+b) − 1] = 0 Support vectors: αᵢ > 0 ↔ yᵢ(w·xᵢ+b) = 1
✅ Conclusion

The optimal hyperplane is uniquely determined by the support vectors alone. The dual QP formulation, solved via SMO or interior-point methods, identifies these support vectors. The solution is globally optimal (convex QP), guaranteeing the maximum margin classifier.

Q5(a) Describe how SVM can be adapted to handle nonlinear classification problems. Discuss the kernel trick. 7M ▼
📖 Kernel Trick

The kernel trick replaces every inner product xᵢᵀxⱼ in the SVM dual formulation with K(xᵢ,xⱼ) = φ(xᵢ)·φ(xⱼ), implicitly computing the inner product in a high-dimensional RKHS without explicitly computing the transformation φ.

∑ Kernel SVM Dual
Replace: xᵢᵀxⱼ → K(xᵢ,xⱼ) in all SVM equations Common Kernels: Linear: K(x,z) = xᵀz Polynomial: K(x,z) = (xᵀz + c)^d RBF/Gauss: K(x,z) = exp(−γ‖x−z‖²) Sigmoid: K(x,z) = tanh(κxᵀz + θ) Mercer's Condition: K valid iff its Gram matrix is PSD
⚙️ Nonlinear SVM Steps
  • Choose appropriate kernel K based on data geometry.
  • Construct kernel matrix Kᵢⱼ = K(xᵢ,xⱼ) — n×n Gram matrix.
  • Solve kernel dual QP to find support vectors and αᵢ.
  • Predict: f(x) = sign(Σ αᵢyᵢK(xᵢ,x) + b) — nonlinear boundary in input space.
✅ Conclusion

The kernel trick is one of ML's most elegant ideas — it enables nonlinear classification at the cost of computing only pairwise similarities. The computational complexity is O(n²) in kernel evaluations, independent of the dimensionality of the feature space φ(x).

Q3(a) Explain the back propagation algorithm with an example and discuss its significance in training multilayer networks. 7M ▼
📖 Backpropagation

Backpropagation is the algorithm for computing gradients of the loss function with respect to all network parameters by applying the chain rule of calculus from output to input layer, enabling gradient descent optimization.

🔧 Backpropagation Algorithm
FORWARD PASS: For each layer l = 1 to L: z^l = W^l · a^(l-1) + b^l [linear combination] a^l = σ(z^l) [activation] COMPUTE LOSS: L = loss(a^L, y) [e.g., MSE or cross-entropy] BACKWARD PASS: Output layer δ^L: δ^L = ∇_a L ⊙ σ'(z^L) For l = L-1, L-2, …, 1: δ^l = (W^(l+1))ᵀ δ^(l+1) ⊙ σ'(z^l) [chain rule] Gradients: ∂L/∂W^l = δ^l · (a^(l-1))ᵀ ∂L/∂b^l = δ^l WEIGHT UPDATE (SGD): W^l ← W^l − η · ∂L/∂W^l b^l ← b^l − η · ∂L/∂b^l
⚙️ Significance
  • Makes training deep networks computationally tractable — O(|weights|) per step.
  • Enables automatic differentiation across any differentiable computation graph.
  • Foundation of all modern deep learning (PyTorch, TensorFlow use autograd).
  • Time complexity per epoch: O(n·W) where W = number of weights.
✅ Conclusion

Backpropagation is the cornerstone of deep learning — it efficiently computes all gradients in O(|W|) time using the chain rule. Its rediscovery by Rumelhart et al. (1986) enabled the modern neural network revolution.

Q4(b) Discuss the steps involved in a Genetic Algorithm, including selection, crossover, and mutation. 8M ▼
📖 Genetic Algorithm

A Genetic Algorithm (GA) is an evolutionary optimization method inspired by Darwinian natural selection that evolves a population of candidate solutions (chromosomes) through selection, crossover, and mutation operators toward high-fitness solutions.

🔧 GA Steps
Step 1 — INITIALIZATION: Generate population P = {s₁, s₂, …, sₙ} [random bit strings] Evaluate fitness f(sᵢ) for each chromosome Step 2 — SELECTION (Survival of fittest): Options: a) Roulette Wheel: P(sᵢ) = f(sᵢ)/Σf(sⱼ) b) Tournament: Select best of k random candidates c) Rank Selection: Rank by fitness, sample by rank Step 3 — CROSSOVER (Reproduction): Select parent pairs (p₁, p₂) Single-point: cut at position k Child₁ = p₁[1..k] + p₂[k+1..n] Child₂ = p₂[1..k] + p₁[k+1..n] Probability of crossover: Pc ≈ 0.6–0.9 Step 4 — MUTATION (Exploration): Flip each bit with probability Pm ≈ 0.001–0.01 Prevents premature convergence Step 5 — REPLACEMENT: Replace old population with offspring (+ elitism) Repeat Steps 2–5 until: max_generations reached OR fitness threshold met
📐 GA Flow
Initialize Population ↓ Evaluate Fitness ←──────────────────┐ ↓ │ Selection (best chromosomes) │ ↓ │ Crossover (combine parents) │ ↓ │ Mutation (random flip) │ ↓ │ New Generation ──────────────────────┘ ↓ Termination? → Output Best Solution
✅ Conclusion

GAs are powerful global optimizers for non-differentiable, multimodal search spaces. The balance between exploitation (selection/crossover) and exploration (mutation) is critical. Elitism (preserving top solutions) prevents regression while maintaining diversity.

Q5(a) Illustrate the process of genetic programming with an example application. 7M ▼
📖 Genetic Programming (GP)

Genetic Programming is an extension of GA where chromosomes are program trees (not fixed-length bit strings), and evolution discovers computer programs that solve a given problem by optimizing their tree structure and operations.

📐 GP Tree Representation
Function set: {+, ×, −, /, sin, cos} Terminal set: {x, y, constants} Program tree for f(x,y) = sin(x) + x·y: [+] / \ [sin] [×] | / \ [x] [x] [y] Crossover: swap subtrees between parent trees Mutation: replace subtree with randomly generated one
⚙️ GP Application Example: Symbolic Regression
  • Task: Find mathematical expression f(x) fitting dataset {(xᵢ,yᵢ)}.
  • Fitness: Minimize MSE = (1/n)Σ(f(xᵢ)−yᵢ)² over training data.
  • Population: 100–1000 random program trees, max depth = 5.
  • Evolution: Tournament selection, subtree crossover (Pc=0.9), point mutation (Pm=0.01).
  • Output: Best program tree = discovered formula (e.g., GP rediscovering x² + 2x).
  • Applications: Feature engineering, robot control, circuit design, game strategy.
✅ Conclusion

GP evolves programs rather than parameters, making it uniquely suited for symbolic regression and automatic program synthesis. Its open-ended search space allows discovering solutions no human would formulate explicitly.

Q1(a) Demonstrate how the Sequential Covering Algorithm works with an example. 7M ▼
📖 Sequential Covering Algorithm

Sequential Covering is a rule induction algorithm that iteratively learns one rule at a time, removes the positive examples covered by that rule, and repeats on the remaining examples until all positives are covered — producing an ordered rule list (decision list).

🔧 Algorithm
Sequential_Covering(Target, Attributes, Examples, threshold): rule_set ← ∅ While positive examples remain AND performance > threshold: rule ← Learn_One_Rule(Target, Attributes, Examples) // Learn_One_Rule: greedy beam search, adding conditions // that maximize accuracy = (TP/(TP+FP)) Remove from Examples all instances covered by rule Add rule to rule_set Return rule_set Learn_One_Rule (FOIL-style): Start: rule = {} [covers all examples] Repeat: Add literal L that most increases FOIL gain: FoilGain(L) = t[log(p₁/(p₁+n₁)) − log(p₀/(p₀+n₀))] Until rule covers no negative examples
⚙️ Example
  • Dataset: PlayTennis (Outlook, Temp, Humidity, Wind → Play?)
  • Iteration 1: Best rule learned: IF Outlook=Sunny AND Humidity=Normal THEN Play=Yes. Remove covered (+) examples.
  • Iteration 2: On remaining: IF Outlook=Overcast THEN Play=Yes. Remove covered.
  • Iteration 3: IF Wind=Weak THEN Play=Yes. Continue until all (+) covered.
  • Result: Ordered rule list classifies all positive examples.
✅ Conclusion

Sequential covering produces human-interpretable rule lists. Rules are ordered by specificity — first matching rule fires. It can handle numerical attributes via discretization and is the foundation of CN2 and RIPPER algorithms.

Q2(a) Illustrate the concept of inverting resolution with an example. 7M ▼
📖 Inverting Resolution

Inverting Resolution is the process of reversing the resolution inference rule to induce general hypotheses from specific examples — it forms the basis of ILP (Inductive Logic Programming) for learning first-order Horn clauses.

∑ Resolution & Inverse Resolution
Forward Resolution (deduction): C₁ = P ∨ L [clause 1] C₂ = Q ∨ ¬L [clause 2, L resolved] ───────────────── C_resolvent = P ∨ Q [conclusion] Inverse Resolution (induction): Given C_resolvent and C₁, infer C₂ Known: P ∨ Q (training example: positive fact) Known: P ∨ L (background knowledge / other clause) Infer: Q ∨ ¬L (hypothesized clause — the rule!)
⚙️ Example — Family Relations
  • Background facts: Parent(Tom, Bob), Parent(Bob, Ann)
  • Positive example: GrandParent(Tom, Ann)
  • Resolution: GrandParent(X,Z) :- Parent(X,Y), Parent(Y,Z) [induced rule]
  • Inverse resolution produces this general rule from specific examples by reversing deduction.
  • Operators: Absorption (induce body), Identification (induce head), Intra-construction (introduce new predicates).
✅ Conclusion

Inverse resolution enables learning of logical rules from examples and background knowledge. It is the theoretical foundation of ILP systems like CIGOL and Progol, which learn relational hypotheses in first-order logic.

Q2(b) Discuss the significance of perfect domain theories in analytical learning. 8M ▼
📖 Perfect Domain Theory

A perfect domain theory is a set of logical axioms/rules that is (1) correct — every statement it entails is true, and (2) complete — it can explain every positive example. Analytical learning (EBL) assumes a perfect domain theory.

⚙️ Significance in Analytical Learning
  • Enables single-example learning: With a perfect theory, one training example suffices to generalize — no need for large datasets.
  • Guides explanation construction: The theory is used to build a proof tree explaining why the example is positive.
  • Guarantees correctness: Generalizations derived from a perfect theory are guaranteed consistent with the domain.
  • Determines generalization: The explanation structure — not the example — defines what features are operationally relevant.
  • Operational criteria: The generalization is restricted to operationally defined concepts (primitive features).

✅ Conclusion

Perfect domain theories enable dramatic sample efficiency — EBL can generalize from a single example. However, real-world domains rarely have perfect theories, motivating hybrid approaches like FOCL that combine analytical and inductive learning.

Q3(a) Summarize Explanation-Based Learning (EBL) and its applications. 7M ▼
📖 EBL Definition

Explanation-Based Learning (EBL) is an analytical learning method that uses domain theory to construct an explanation (proof tree) of a training example, then generalizes that explanation to form an operational concept definition applicable to new instances.

🔧 EBL Algorithm
Input: Training example, Target concept, Domain theory, Operationality criteria Step 1 — EXPLAIN: Construct proof tree proving: Domain_Theory ⊢ Target_Concept(training_example) [Use backward chaining with domain axioms] Step 2 — GENERALIZE: Replace specific constants with variables in proof tree Preserve structure of explanation Result: Generalized proof schema = Learned Rule Step 3 — OPERATIONALIZE: Ensure all leaf nodes of proof tree are operational (Observable, computable predicates) Output: New rule — if operational leaves hold → target concept
⚙️ Applications
  • Game Playing: Chess — EBL learns macro-operators (e.g., "safe king move") from successful game analysis.
  • Planning: Robots learn operational plans by explaining successful demonstrations.
  • Medical Diagnosis: Generalize specific patient case using medical domain theory.
  • Theorem Proving: Caching useful lemmas derived during proof search.
  • Natural Language: Generalizing parse structures from example sentences.
✅ Conclusion

EBL learns efficiently from single examples by leveraging domain knowledge — eliminating the need for large training sets. Its limitation is sensitivity to domain theory quality: imperfect theories lead to incorrect generalizations, motivating hybrid FOCL.

Q4(b) Describe the Q-Learning algorithm and its role in reinforcement learning. 8M ▼
📖 Q-Learning

Q-Learning (Watkins, 1989) is a model-free, off-policy RL algorithm that learns the optimal action-value function Q*(s,a) — the expected cumulative discounted reward from state s taking action a — directly from experience without needing a model of the environment.

∑ Q-Learning Update Rule
Q-value: Q(s,a) = E[Σ_{t=0}^∞ γᵗ rₜ | s₀=s, a₀=a, π*] TD Update (after transition s,a,r,s'): Q(s,a) ← Q(s,a) + α [r + γ·max_{a'} Q(s',a') − Q(s,a)] └────────────────────────────────┘ TD Error (Temporal Difference) Where: α ∈ (0,1]: learning rate γ ∈ [0,1]: discount factor (0=myopic, 1=far-sighted) r: immediate reward max_{a'}Q(s',a'): bootstrap from best next state value
🔧 Q-Learning Algorithm
Initialize Q(s,a) ← 0 for all s,a [or random] For each episode: s ← initial state While s is not terminal: a ← ε-greedy: random w.p. ε, else argmax_a Q(s,a) Execute action a in environment Observe reward r and next state s' Q(s,a) ← Q(s,a) + α[r + γ·max_{a'}Q(s',a') − Q(s,a)] s ← s' Decay ε (exploration rate) over time Output: Q-table (optimal policy: π*(s) = argmax_a Q(s,a))
✅ Conclusion

Q-learning is guaranteed to converge to Q* under mild conditions (visiting all state-action pairs infinitely often). It is off-policy (learns Q* regardless of the behavior policy). DQN (Deep Q-Network) extends Q-learning to large state spaces using neural networks as function approximators.
